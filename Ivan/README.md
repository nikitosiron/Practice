# TravelLine Tech — мини-конструктор сайта

Учебный проект летней практики. Публичная страница + админ-панель для управления контентом сайта travelline.tech.

## Стек технологий

- HTML, CSS, JavaScript (vanilla)
- Node.js, Express
- JSON-файл как хранилище данных

## Запуск

```bash
cd Practice/Ivan
npm install
npm start
```

## Адреса

- Публичная страница: http://localhost:3000/
- Админ-панель: http://localhost:3000/admin

## Управляемые блоки

| Блок | Описание |
|------|----------|
| О нас (Hero) | Заголовок и ключевые показатели |
| Сотрудники | CRUD, поиск, карточки/таблица, активация/деактивация |
| Должности | CRUD, выбор из списка при редактировании сотрудника, inline-добавление |
| Таймлайн платформы | CRUD, год/название/аудитория (B2B, B2C, B2E)/описание, активация |
| Вакансии | CRUD, поиск, активация/деактивация |
| Плюшки | CRUD, активация/деактивация |
| Фотогалерея | CRUD (фото/видео), активация, массовое удаление |
| Логотипы партнёров | CRUD, активация |
| Офисы («Работай как удобно») | CRUD, активация |
| Направления | CRUD, вложенный редактируемый список технологий, активация |
| Контактная форма | Форма-singleton: заголовок, описание, список направлений выпадающего списка |

## Хранение данных

Все данные хранятся в `data/data.json`. Структура:

```json
{
  "hero": { "title": "...", "stats": [] },
  "team": [],
  "vacancies": [],
  "benefits": [],
  "positions": [],
  "gallery": [],
  "timeline": [],
  "brands": [],
  "work": [],
  "directions": [],
  "contactForm": { "title": "...", "description": "...", "directions": [] }
}
```

Неактивные записи (`active: false`) отображаются в админке, но скрыты на публичной странице. Контактная форма — единственный объект (не список), поэтому у неё нет поля `active` и раздел в админке — это форма, а не карточки.

## Тесты

```bash
npm test
```

36 тестов покрывают: чтение/запись данных, CRUD для всех разделов (сотрудники, должности, вакансии, плюшки, галерея, таймлайн, партнёры, офисы, направления, контактная форма), совместимость со старыми данными без поля `active`, фильтрацию неактивных записей, валидацию URL вакансий, валидацию типа файла в галерее (image/video), валидацию года и категории таймлайна, валидацию вложенных технологий у направлений.
